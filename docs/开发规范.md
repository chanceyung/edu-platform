# 开发规范（DEVELOPMENT）

> 本文件是 `CLAUDE.md` 的展开，规定具体技术执行细节。与 `CLAUDE.md` 冲突时以 `CLAUDE.md` 为准。

## 一、后端分层规范（严格）

每个业务模块统一分层：
```
controller/    只做：参数校验、权限注解、调用 service、组装 VO 返回。禁止业务逻辑、禁止直连 DB。
service/       业务逻辑。接口 + impl 分离。
mapper/        MyBatis-Plus Mapper，只做持久化。
entity/        数据库实体（@TableName）。
dto/           入参对象（带校验注解）。
vo/            出参对象（返回前端）。
bo/            业务中间对象（跨 service 传递）。
convert/       对象转换（MapStruct），禁止实体直接当 VO 返回。
config/        模块配置。
```

- **禁止**：Controller 写业务、Service 写 SQL 跨表 join 业务、Entity 暴露给前端。
- 事务注解 `@Transactional` 只标在 Service 方法，且只标注需要事务的方法（不滥用）。

## 二、命名规范

| 类型 | 规则 | 示例 |
|---|---|---|
| 类 | PascalCase，模块前缀 | `OrgStudentService`、`ExamPaperController` |
| 方法/变量 | camelCase | `listByClassId` |
| 常量/枚举 | UPPER_SNAKE | `EXAM_STATUS_FINISHED` |
| 包 | 全小写单数 | `com.eduplatform.schoolorg` |
| 数据库表 | 模块前缀 + snake | `org_student`、`qbank_question`、`ai_task_log` |
| 数据库字段 | snake_case | `create_time`、`tenant_id` |
| API 路径 | 小写连字符/复数 | `/api/v1/org/students` |

## 三、API 规范

- 版本前缀 `/api/v1`，模块二级路径 `/api/v1/{module}/...`
- 统一响应：`{ "code": 0, "message": "success", "data": {...} }`，错误码集中定义
- 入参用 `@Valid` + DTO 校验注解（`@NotNull`/`@Size` 等）
- 分页统一 `PageDTO{page,size}` → `PageVO{total,records}`
- 权限：`@PreAuthorize` 或自定义数据权限注解（教师只见自己班）
- 文档：Knife4j 注解（`@Operation`/`@Schema`），与代码同步

## 四、数据库规范

- 每张表必有字段：`id`(bigint unsigned, 主键)、`tenant_id`(varchar)、`create_time`/`update_time`(datetime)、`creator`/`updater`(varchar)、`is_deleted`(tinyint, 软删)
- 主键策略：雪花算法或自增（按模块统一）
- 索引：高频查询字段建索引；联合索引注意最左前缀
- 枚举值用 tinyint + 数据字典表，代码里用枚举类映射，禁止魔法数字
- 时间用 datetime；金额/分值用千分制 int（score×1000）
- 富文本统一外置 `sys_text_content(id, content)`，业务表存 `*_text_content_id`
- **所有变更走 Flyway 迁移脚本**（`V{日期}__{描述}.sql`），禁止手改库；脚本进版本控制

## 五、前端规范（Vue3 + TS）

- 组件 PascalCase，`<script setup lang="ts">`
- 状态管理：Pinia；请求：axios 封装（统一拦截/错误码处理）
- 路由按模块分组，权限路由动态加载
- API 调用集中在 `api/` 目录，类型定义在 `types/`
- 样式：scoped + 设计令牌（颜色/间距统一），禁止硬编码样式覆盖
- 表单：统一校验规则；表格：统一封装组件
- TypeScript strict 模式，禁止 `any` 滥用

## 六、移动端规范（uniapp）

- 学生端/家长端共用 uniapp 工程，条件编译区分平台
- 核心页面 H5 + 小程序双端兼容；APP 按需
- 家长端与学生端逻辑隔离，权限严格

## 七、AI / Agent 规范（详见 `ai-platform/README.md`）

- 模型调用统一经 `ai-service` 的智谱网关，禁止业务模块直连智谱 API
- 模型分工：复杂长程→GLM-5.2，高频简单→GLM-4 Air，识图→GLM-5V，生图→CogView，视频→CogVideoX，向量→Embedding-3
- **每个 AI 调用必写 `ai_*_log`**（输入摘要/输出/模型/token/耗时/操作人/可回滚标识）
- Agent 必须实现信任内核（动作分级/人在回路/可追溯/可回滚/可中断）
- Prompt 模板版本化管理（`ai-platform/prompts/`），变更需评测回归
- 评测集：批改/组卷/评语等核心 AI 能力有评测集，回归不达标阻塞发布

## 八、安全规范

- 密码：BCrypt 加盐；敏感数据脱敏返回
- 越权防护：所有查询带租户/数据权限过滤（RBAC + 数据权限拦截）
- 输入校验防 XSS/SQL 注入（参数化查询、富文本过滤）
- 接口限流（Redis）、防刷
- 凭据/密钥用配置中心或环境变量，**严禁入库**（CI 扫密钥）
- 审计日志：敏感操作（删数据/改成绩/发全校通知）必记操作日志

## 九、测试规范

- 单元测试：JUnit5 + Mockito，命名 `should_xxx_when_yyy`
- 集成测试：Testcontainers（真实 MySQL/Redis）
- 覆盖率：核心模块 ≥80%，其他 ≥60%（Jacoco，CI 卡门槛）
- 前端：Vitest（组件逻辑）+ 关键流程 E2E（Playwright）
- AI：评测集 + 回归（准确率/幻觉率指标）
- 测试数据用工厂/夹具，不依赖共享脏数据

## 十、Git 与协作

- 分支：`main`/`develop`/`feature-{模块}-{简述}`/`fix-{简述}`/`release-{版本}`
- Commit：Conventional Commits + 模块前缀，例 `feat(ai-service): GLM-5.2 批改Agent信任内核`
- 一个 PR 一件事，粒度小、可回滚
- PR 模板：改了什么/为什么/怎么测/影响面
- CI 必绿 + 1 人 Review 才可合并；`main` 保护分支

## 十一、性能与稳定

- 慢查询监控（>500ms 告警）；N+1 问题 CI 检测
- 缓存：热点数据 Redis，注意一致性与失效
- 异步：批改/通知/统计等用消息队列异步，不阻塞主流程
- 高并发考试：Redis 缓存 + 交卷队列削峰
- 限流降级：AI 服务异常时降级（人工/规则），保核心可用
